# 智能客服 Agent 使用说明

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置 API Key

#### 方式:使用 config.json 文件(推荐)

编辑 `config.json` 文件,将 `api_key` 字段替换为您的实际 API Key:

```json
{
  "api_key": "your_actual_api_key_here",
  "base_url": "https://dashscope.aliyuncs.com/compatible-mode/v1",
  "model": "deepseek-v4-pro",
  "embedding_model": "text-embedding-v3"
}
```

### 3. 准备知识库内容

编辑 `knowledge_base/` 目录下的文件,填入您的业务知识内容。

格式建议:
- 使用 `##` 分隔不同主题
- 每个主题下用段落详细说明
- 可以包含:公司介绍、产品信息、常见问题、联系方式等

### 4. 启动服务

```bash
python app.py
```

服务将在 http://localhost:5000 启动。

## API 接口

### 1. 检查服务状态

**请求:**
```
GET /start
```

**响应示例:**
```json
{
  "status": "ready",
  "message": "客服系统已就绪",
  "assistant_id": "asst_xxxxx",
  "knowledge_base": true
}
```

### 2. 发送对话请求

**请求:**
```
POST /chat
Content-Type: application/json

{
  "message": "你们有哪些产品?",
  "session_id": "optional_session_id"
}
```

**响应示例:**
```json
{
  "reply": "我们提供以下产品:\n1. 智能办公系统...\n2. 数据分析平台...\n3. 客户关系管理系统...",
  "tool_calls": [],
  "session_id": "session_id_here",
  "finished": false
}
```

## 测试示例

### 使用 curl 测试

```bash
# 检查服务状态
curl http://localhost:5000/start

# 发送对话请求
curl -X POST http://localhost:5000/chat \
  -H "Content-Type: application/json" \
  -d "{\"message\": \"你好,我想了解一下你们的产品\"}"
```

### 使用 Python 测试

```python
import requests

# 检查服务状态
response = requests.get('http://localhost:5000/start')
print(response.json())

# 发送对话
response = requests.post(
    'http://localhost:5000/chat',
    json={
        'message': '你好,我想了解一下你们的产品',
        'session_id': 'test_session_001'
    }
)
print(response.json())

# 继续对话(使用相同的 session_id)
response = requests.post(
    'http://localhost:5000/chat',
    json={
        'message': '产品A的价格是多少?',
        'session_id': 'test_session_001'
    }
)
print(response.json())
```

## 常见问题

### 1. API Key 配置错误

**错误信息:** `Authentication error` 或 `Invalid API key`

**解决方法:**
- 检查 API Key 是否正确
- 确认 config.json 文件配置正确

### 2. 知识库向量化失败

**错误信息:** `生成向量失败`

**解决方法:**
- 检查网络连接
- 确认 API Key 有 embedding 模型权限
- 检查 db 目录是否有写入权限

### 3. Airtable 提交失败

**错误信息:** `提交失败: Not authenticated`

**解决方法:**
- 检查 Airtable API Token 是否正确
- 确认 Base ID 和 Table Name 正确
- 验证 Token 权限设置

### 4. 助手创建失败

**错误信息:** 创建助手时出错

**解决方法:**
- 检查 config.json 中的模型名称是否正确
- 确认 API Key 有创建助手的权限

## 自定义配置

### 修改助手指令

编辑 `modules/prompt/__init__.py` 文件中的 `CUSTOMER_SERVICE_PROMPT` 常量,可以自定义助手的行为和回复风格。

### 添加工具

在 `modules/tools/` 目录中添加工具定义和执行函数,扩展助手能力。

### 更新知识库

修改 `knowledge_base/` 目录下的文件后,删除 `db/vector_data.json`,重启服务即可重新向量化。

## 部署建议

### 生产环境部署

1. 使用 gunicorn 代替 Flask 开发服务器:
```bash
pip install gunicorn
gunicorn -w 4 -b 0.0.0.0:5000 app:app
```

2. 配置 Nginx 反向代理

3. 使用 HTTPS

4. 配置日志记录

5. 设置进程管理器(systemd/supervisor)

## 技术支持

如遇到问题,请检查:
1. Python 版本(建议 3.8+)
2. 依赖包版本
3. 配置文件格式
4. 网络连接
5. 日志输出
